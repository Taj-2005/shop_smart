# Software Development Life Cycle (SDLC)

This folder holds **process, planning, and quality** documentation for the project.

**Primary SDLC document:** [sdlc.md](./sdlc.md) — phases, RACI, Definition of Done, development workflow.

## Purpose

- Describe **how** the team builds and ships ShopSmart (phases, roles, checkpoints).
- Satisfy coursework expectations around **SDLC usage**, traceability, and **MVP** scope.
- Capture **Definition of Done**, milestones, and demo readiness.

## What to add here

| Document | Description |
|----------|-------------|
| Process overview | Phases (design → build → test → release) and entry/exit criteria |
| RACI / roles | Who owns architecture, backend, frontend, data, QA |
| Traceability | Simple matrix: requirement → feature → test → diagram (optional table) |
| MVP scope | Explicit “in scope for submission” vs future work |
| QA checklist | Manual checks before demo or release |
| Demo script | Step-by-step narrative for presentation (5–10 minutes) |

## Conventions

- Use **dates and version** in filenames when documents are milestone-specific (e.g. `MILESTONE-1.md`).
- Keep **one source of truth** for deadlines; avoid duplicating dates across many files without a pointer.
