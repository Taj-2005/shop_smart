System overview

The browser loads the Next.js app from the frontend origin and calls the API origin with JSON over HTTPS. JWT access tokens are stored in localStorage and sent as Authorization: Bearer; refresh uses POST /api/auth/refresh with a JSON body.

High-level request flow

        │  Axios — Authorization: Bearer <access>

Client-side auth

Tokens live in localStorage (namespaced keys). Axios attaches Authorization: Bearer; on 401 the client calls POST /api/auth/refresh with { refreshToken }, persists the new session, and retries once.

sequenceDiagram participant SPA as Next.js client participant API as Express API participant User as User User->>SPA: Action SPA->>API: Request + Bearer access alt 200 OK API-->>SPA: JSON SPA-->>User: UI update else 401 API-->>SPA: 401 SPA->>API: POST /auth/refresh + body refreshToken alt Refresh OK API-->>SPA: New access + refresh + user SPA->>SPA: persist localStorage SPA->>API: Retry original request else Refresh fail SPA->>SPA: clear session SPA-->>User: Redirect login end end
  • ProtectedRoute, AdminRoute, and SuperAdminRoute gate pages using auth context.
  • API enforces FRONTEND_URL for CORS.

SOLID (summary)

  • S — Focused services, repositories, and middleware.
  • O — New pricing profiles and notification channels via extension points.
  • L — Controllers substitutable through BaseController.
  • I — Small interfaces (e.g. reader vs writer splits).
  • D — High-level modules depend on abstractions (ITokenService, etc.).
